# Add a Rendered / Plain-text toggle to the diff view

## Problem

The diff tab currently only offers a "rendered" view of the comparison (colored +/- lines, in either unified or side-by-side layout — see `002-diff-layout-toggle-icon-only.md` for that toggle). There's no way to see the raw diff text itself without leaving the app (the closest is the "Save Diff" button, which writes the raw text to disk but doesn't display it).

## Required change

Add a second, independent toggle to the diff toolbar (`Home.razor`, `.diff-toolbar-actions` block, alongside the layout toggle and Save Diff button):

- **Rendered** — the current colored HTML view (unified or side-by-side, per the existing layout toggle).
- **Plain text** — the raw unified-diff text, unstyled, in a `<pre>` block. This is the exact same text already generated by `DiffService.BuildUnifiedDiffText` and stored on `MarkdownDocument.Content` for the Save Diff button — no new diff-generation logic needed, just render `TabService.ActiveDocument.Content` directly.

### Model

Add a new mutable field to `MarkdownDocument` (`wahlberg.maui/Models/MarkdownDocument.cs`), alongside the existing `DiffShowSideBySide`:

```csharp
public bool DiffShowPlainText { get; set; }
```

This keeps the mode remembered per-tab the same way `DiffShowSideBySide` already is (state lives on the document instance, so switching tabs away and back preserves it).

### UI

- New icon-only toggle button (follow the pattern from `002-diff-layout-toggle-icon-only.md`) — e.g. `bi-code-slash` for switching to "Plain text", `bi-file-richtext` for switching back to "Rendered" (exact icon choice is not critical — pick whatever reads clearly, `title` attribute carries the label either way).
- Add a `ToggleDiffPlainText()` handler in `Home.razor`'s `@code` block, mirroring `ToggleDiffLayout()`:
  ```csharp
  private void ToggleDiffPlainText()
  {
      var doc = TabService.ActiveDocument;
      if (doc is null || !doc.IsDiff) return;
      doc.DiffShowPlainText = !doc.DiffShowPlainText;
      StateHasChanged();
  }
  ```
- When `DiffShowPlainText` is true, the Unified/Side-by-side layout toggle doesn't apply (plain text has no layout concept) — hide it or disable it while plain text mode is active. Pick whichever is simpler to implement cleanly; either is acceptable.

### Rendering

In the diff content branch of `Home.razor` (where `TabService.ActiveDocument.IsDiff` is true), branch further on `DiffShowPlainText`:

```razor
@if (TabService.ActiveDocument.DiffShowPlainText)
{
    <pre class="diff-plain-text">@TabService.ActiveDocument.Content</pre>
}
else
{
    <div class="diff-view">
        @((MarkupString)(TabService.ActiveDocument.DiffShowSideBySide ? TabService.ActiveDocument.DiffSideBySideHtml : TabService.ActiveDocument.DiffUnifiedHtml))
    </div>
}
```

Add a `.diff-plain-text` rule in `wahlberg.maui/wwwroot/app.css` near the other `.diff-*` rules — monospace font (reuse `var(--theme-code-font)`), matching padding/overflow behavior to `.diff-view`, no color coding.

## Verification

1. Open a diff tab.
2. Toggle to Plain text — confirm the raw diff text renders unstyled (no red/green), matching exactly what Save Diff would write to a `.diff` file.
3. Toggle back to Rendered — confirm it returns to whichever layout (unified/side-by-side) was active before switching to plain text.
4. Switch to a different tab and back — confirm the plain-text/rendered choice for this diff tab is remembered.
