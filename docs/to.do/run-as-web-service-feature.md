# run as web service feature

it's frequently come up that you aren't able to test this easily without taking over the PC (and sometimes i accidentally interfere with this when i don't realise you are busy with that). Can we add a feature (ideally a commandline argument which accepts an optional port number) that runs wahlberg as a service, allowing it to be used within a browser window?

aside from potentially being useful to end users (arguably this would even allow it to be used on linux), it seems like you would be able to do a lot of testing through your own browser.

i think this should take the highest priority, since it would improve your ability to test other features in the todo list
